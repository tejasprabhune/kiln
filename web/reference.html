KILN

A Cargo-style CLI for SystemVerilog

Overview Docs Reference GitHub crates.io docs.rs

[package]

KeyTypeNotes
name required string Valid SystemVerilog identifier: starts with a letter or _, followed by letters, digits, or _.
version required string Semver string, e.g. "0.1.0".
authors optional list of strings Free-form. E.g. ["Jane <jane@example.com>"].
description optional string Single-line description.
license optional string SPDX expression, e.g. "MIT OR Apache-2.0".

[design]

KeyTypeDefault
top required string
sources optional list of glob strings ["src/**/*.sv", "src/**/*.svh", "src/**/*.v"]
include_dirs optional list of paths []
defines optional string-to-string map {}

Globs are evaluated relative to the directory containing Kiln.toml. include_dirs entries must exist on disk when the manifest is loaded (the check is skipped during kiln new). defines entries with an empty string value become valueless +define+FOO flags; non-empty values become +define+FOO=bar. 

[design]
top          = "my_top"
sources      = ["rtl/**/*.sv", "rtl/**/*.v", "ip/**/*.sv"]
include_dirs = ["rtl/include", "ip/include"]
defines      = { SIMULATION = "", WIDTH = "8" }

[dependencies]

Each entry is a named dependency in one of three forms:

[dependencies]
# git dep with semver constraint (matched against repo tags)
axi = { git = "https://github.com/pulp-platform/axi.git", version = "0.39" }

# git dep pinned to a specific tag or commit SHA
common_cells = { git = "https://github.com/pulp-platform/common_cells.git", rev = "v1.32.0" }

# path dep relative to the project root (or absolute)
local_ip = { path = "../local_ip" }

Resolved versions are written to Kiln.lock, which should be committed. Use kiln update to refresh it.

[lint]

Per-diagnostic severity overrides. Keys are slang's diagnostic option names (the same strings passed to -W<name> on the slang CLI). Values are "error", "warn", or "allow". 

[lint]
no-timescale-unit    = "allow"   # suppress missing-timescale warnings
width-trunc          = "error"   # promote truncation warnings to errors
unused-net           = "warn"
implicit-net         = "allow"

Common diagnostic names

NameDefaultWhat it covers
no-timescale-unitwarnA design element lacks a `timescale directive while others in the design have one. Suppress this if your simulator sets a default timescale.
width-truncwarnImplicit truncation when assigning a wider value to a narrower type.
width-expandwarnImplicit zero- or sign-extension when assigning to a wider type.
implicit-netwarnNet used without an explicit declaration (relies on implicit-wire rules).
unused-netwarnNet declared but never driven or read.
unused-variablewarnVariable declared but never written or read.
unused-importwarnPackage imported with import pkg::* but nothing from it is used.
unused-typedefwarnTypedef declared but never referenced.
unused-paramwarnParameter declared but never referenced.
always-comb-blockingwarnBlocking assignment (=) inside always_comb.
always-ff-blockingwarnBlocking assignment (=) inside always_ff.
case-incompletewarncase statement does not cover all possible values and has no default.
cast-truncateswarnExplicit cast that results in truncation.
sign-comparewarnComparison between signed and unsigned values.
empty-bodywarnEmpty body in an if, while, or similar construct.
port-coercionwarnType coercion at a port connection boundary.
unconnected-portwarnInstance port left unconnected without .* or explicit .port().
missing-porterrorRequired port in an instance connection is missing entirely.

Any diagnostic name accepted by slang -W<name> is valid here. Run slang --help-diagnostics for the full list. If you're unsure of the key for a specific warning, kiln check now prints it in brackets after the message — e.g. warning: ... [no-timescale-unit] — so you can copy it directly into [lint]. Setting a rule to "error" or "warn" also passes -W<name> to slang so diagnostics slang would normally suppress at the default level are surfaced. "allow" is post-filtering: the diagnostic still reaches kiln but is dropped before rendering.

[wave]

KeyTypeDefault
format optional "fst" | "vcd" "fst"
enabled_by_default optional bool false

When enabled_by_default = true, every kiln test run behaves as if --trace was passed. FST is smaller and faster than VCD; prefer it unless your waveform viewer requires VCD. 

[wave]
format             = "fst"
enabled_by_default = false

Complete example

[package]
name        = "counter"
version     = "0.1.0"
authors     = ["Tejas Prabhune <prabhune@berkeley.edu>"]
description = "A parameterized up/down counter"
license     = "MIT OR Apache-2.0"

[design]
top          = "counter"
sources      = ["src/**/*.sv"]
include_dirs = ["include"]
defines      = { SIMULATION = "" }

[dependencies]
common_cells = { git = "https://github.com/pulp-platform/common_cells.git", version = "1.32" }

[lint]
no-timescale-unit    = "allow"
width-trunc          = "error"
unused-net           = "warn"
implicit-net         = "allow"

[wave]
format             = "fst"
enabled_by_default = false
